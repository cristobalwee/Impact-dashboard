  Total budget: ~90 min (analysis is the core deliverable; UI is iterated in

  0. Definition of impact                                                       
                                                                                
  Impact is a weighted blend of five legible pillars. No single axis can        
  dominate, every score decomposes into plain-English reasons, and the weights  
  are user-adjustable in the UI. Reweighting must recompute rankings            
  client-side from stored pillar scores — never require re-running the data     
  pipeline.                                                                     
                                                                                
  Default weights (sum = 100, exposed as sliders):                              
                                                                                
  PillarWeightOne-line meaningAuthorship value25Durable, meaningful code (not   
  raw volume)Collaboration & leverage25How much you enable othersOwnership &    
  criticality20How much load-bearing code you ownConsistency15Sustained         
  engagement, not one big dumpInfluence / centrality15Your position in the      
  collaboration graph                                                           
                                                                                
                                                                                
  1. Rubric — exact metrics                                                     
                                                                                
  Cohort: engineers with ≥3 commits OR ≥3 reviews OR ≥2 merged PRs in the       
  90-day window. If fewer than 8 qualify, lower the commit/review threshold to  
  2 and note it in meta.                                                        
                                                                                
  Normalization: every raw metric → percentile rank within the cohort (0–100),  
  robust to outliers. Pillar scores combine percentiles as below. Composite =   
  Σ(pillar × weight) / Σweight.                                                 
                                                                                
  Authorship value                                                              
                                                                                
  effective_churn   = insertions + deletions on non-generated, non-vendored     
  files                                                                         
                      (deletions counted POSITIVE)                              
  weighted_churn    = effective_churn × dir_weight   (source 1.0, tests 0.7,    
  docs/config 0.4)                                                              
  survival_rate     = surviving authored lines at HEAD / lines authored in      
  window  (git blame)                                                           
  authored_raw      = weighted_churn × (0.5 + 0.5 × survival_rate)              
  Authorship pillar = 0.7 × pct(authored_raw) + 0.3 × pct(merged_pr_count)      
                                                                                
  Collaboration & leverage                                                      
                                                                                
  reviews_substantive = reviews that are CHANGES_REQUESTED, or                  
  APPROVED/COMMENTED with a non-empty body                                      
                        (exclude empty rubber-stamp approvals)                  
  distinct_reviewed   = number of distinct PR authors this person reviewed      
  discussion          = review_comments + issue_comments                        
  coauthors           = distinct Co-authored-by partners                        
  Collab pillar = 0.4·pct(reviews_substantive) + 0.3·pct(distinct_reviewed)     
                + 0.2·pct(discussion) + 0.1·pct(coauthors)                      
                                                                                
  Ownership & criticality                                                       
                                                                                
  For each file at HEAD: primary_author = engineer with the most surviving      
  lines.                                                                        
  owned_files     = files where this engineer is primary AND owns ≥50%          
  surviving lines                                                               
  file_criticality= # of distinct team commits touching that file in the last   
  365 days  (hotness)                                                           
  ownership_raw   = Σ over owned_files ( surviving_line_share ×                 
  file_criticality )                                                            
  bus_factor_raw  = Σ file_criticality over files where engineer owns ≥80%      
  surviving lines                                                               
  Ownership pillar = 0.7·pct(ownership_raw) + 0.3·pct(bus_factor_raw)           
                                                                                
  Consistency                                                                   
                                                                                
  active_weeks    = distinct ISO weeks (cap 13) in the 90d window with ≥1       
  commit/PR/review                                                              
  recency_weight  = each active week weighted by 1 + (week_index /              
  total_weeks)×0.5                                                              
  consistency_raw = Σ recency-weighted active weeks                             
  Consistency pillar = pct(consistency_raw)                                     
                                                                                
  Influence / centrality                                                        
                                                                                
  Build a graph: edge A→B when A reviews B's PR or co-authors with B (90d       
  window).                                                                      
  centrality = weighted degree (in+out). Use networkx betweenness if available; 
  else weighted degree.                                                         
  Influence pillar = pct(centrality)                                            
                                                                                
                                                                                
  2. Data hygiene (part of the rubric, not optional)                            
                                                                                
                                                                                
  Bots: drop dependabot[bot], github-actions[bot], posthog-bot, renovate[bot],  
  *[bot], and any author whose login ends in bot or matches a maintained

  Identity merge: generate/extend a .mailmap; merge on shared email or shared

Maintain the glob list in hygiene.py.
